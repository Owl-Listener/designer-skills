# plan-sprint
Plan a design sprint.
## Steps
1. **Frame the challenge** — Define the sprint question and scope using the design-sprint-plan skill (see skills/design-ops--design-sprint-plan.md).
2. **Assemble the team** — Identify roles and participants using the team-workflow skill (see skills/design-ops--team-workflow.md).
3. **Plan activities** — Structure each day's activities using the design-sprint-plan skill (see skills/design-ops--design-sprint-plan.md).
4. **Prepare materials** — Define prototyping approach and testing plan.
5. **Recruit testers** — Plan user testing sessions for the final day.
6. **Set review criteria** — Define how sprint outcomes will be evaluated using the design-review-process skill (see skills/design-ops--design-review-process.md).
## Output
Complete sprint plan with challenge statement, team roster, daily schedules, materials list, testing plan, and success criteria.
Consider following up with the handoff workflow after the sprint to document the winning concept.
