# frame-problem
Structure an ambiguous challenge into a clear, actionable problem definition.
## Steps
1. **Explore** — Unpack the challenge, identify assumptions and unknowns.
2. **Stakeholders** — Map affected parties using the stakeholder-alignment skill (see skills/ux-strategy--stakeholder-alignment.md).
3. **Define** — Write clear problem statement using the design-brief skill (see skills/ux-strategy--design-brief.md).
4. **Constrain** — Identify technical, business, design constraints.
5. **Success criteria** — Define evaluation criteria using the metrics-definition skill (see skills/ux-strategy--metrics-definition.md).
6. **Principles** — Set decision-making principles using the design-principles skill (see skills/ux-strategy--design-principles.md).
7. **Prioritize** — If multiple sub-problems, rank using the opportunity-framework skill (see skills/ux-strategy--opportunity-framework.md).
## Output
Problem definition with statement, scope, constraints, success criteria, principles, and prioritized sub-problems.
Consider following up with the strategize workflow or the benchmark workflow.
