# setup-workflow
Set up a design team workflow.
## Steps
1. **Team structure** — Define roles and responsibilities using the team-workflow skill (see skills/design-ops--team-workflow.md).
2. **Rituals** — Establish collaboration cadence using the team-workflow skill (see skills/design-ops--team-workflow.md).
3. **Critique process** — Set up design critique format using the design-critique skill (see skills/design-ops--design-critique.md).
4. **Review gates** — Define quality checkpoints using the design-review-process skill (see skills/design-ops--design-review-process.md).
5. **Versioning** — Establish file and library versioning using the version-control-strategy skill (see skills/design-ops--version-control-strategy.md).
6. **QA process** — Set up design QA workflow using the design-qa-checklist skill (see skills/design-ops--design-qa-checklist.md).
## Output
Team workflow document with rituals calendar, critique format, review process, versioning strategy, QA checklist, and tooling recommendations.
Consider following up with the plan-sprint workflow to kick off your first project with the new workflow.
